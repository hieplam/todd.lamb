## Goal

Provide locale-keyed UI string translation loaded at build time so all user-visible text in the blog interface can be internationalized without code duplication, currently defaulting to English.

## Parent Fit

| Field | Value |
| --- | --- |
| Container | Hiep Lam Blog (c3-1) |
| Category | Feature (11) |
| Owned files | src/i18n/index.ts, src/i18n/format.ts, src/i18n/types.ts, src/i18n/lang/en.ts |
| Depended on by | pages-routing and layouts call useTranslations() for user-visible strings |

## Purpose

Owns the translation registry (index.ts using import.meta.glob to load lang/*.ts modules), the UIStrings type contract (types.ts), the template string interpolation utility (format.ts/tplStr), and the English locale file (lang/en.ts). Does NOT own astro.config.ts i18n routing configuration which is a framework-level concern.

## Foundational Flow

| Aspect | Detail | Reference |

| State | translations registry is module-level constant populated from eager glob at import time | c3-1 |
| Shared deps | astro.config.ts i18n.locales config defines valid locales; fallback to en if locale missing | c3-1 |

## Business Flow

| Aspect | Detail | Reference |
| --- | --- | --- |
| Primary path | Component calls useTranslations(locale), receives UIStrings record, accesses t.key for each string | c3-1 |
| Fallback | If requested locale has no translation, useTranslations returns English strings | c3-1 |
| Template strings | tplStr(template, vars) used for strings with dynamic values like post count or author name | c3-1 |
| New locale | Add new lang/xx.ts implementing UIStrings, it is automatically discovered by import.meta.glob | c3-1 |

## Contract

| Surface | Direction | Contract | Boundary | Evidence |
| --- | --- | --- | --- | --- |
| useTranslations | OUT | Returns UIStrings typed record for the given locale or falls back to English | All page templates and components | src/i18n/index.ts |
| UIStrings type | OUT | TypeScript interface defining all translatable keys; callers must use typed access | src/i18n/types.ts consumers | src/i18n/types.ts |
| tplStr | OUT | Interpolates template string with named variable map and returns final string | Callers needing dynamic string values | src/i18n/format.ts |

## Change Safety

| Risk | Trigger | Detection | Required Verification |
| --- | --- | --- | --- |
| Missing key in new locale | Adding a key to UIStrings type without updating all lang/*.ts files | TypeScript error in lang files that don't implement the new key | src/i18n/types.ts change + astro check |
| tplStr interpolation mismatch | Changing variable names in template strings without updating callers | Runtime rendering of raw {variableName} in built page | src/i18n/format.ts change + manual UI review |
